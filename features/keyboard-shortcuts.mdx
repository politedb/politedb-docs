description: "Work quickly in PoliteDB without leaving the keyboard"

PoliteDB includes a core set of keyboard shortcuts so you can navigate, run actions, and manage tabs quickly.

Shortcuts are configurable, so you can keep the defaults or adapt them to your workflow.

## Default shortcuts

By default, PoliteDB uses these bindings (`Mod` = `Cmd` on macOS, `Ctrl` on Windows/Linux):

- **Open search**: `Mod+K` (and `Mod+P` is also supported in connection workspace).
- **New SQL editor**: `Mod+T`.
- **Save changes**: `Mod+S`.
- **Refresh**: `Mod+R`.
- **Close current tab/window**: `Mod+W`.

These actions are context-aware:

- **Close current** closes the active SQL/table window first. If no window is active, it closes the current connection tab.
- **Save changes** triggers the current save flow (for example, pending table edits or current SQL draft workflow).
- **Open search** opens the database object search dialog in the active connection context.

## Editing and input behavior

When your cursor is inside an editable control (`input`, `textarea`, or content-editable area), PoliteDB intentionally limits shortcut handling to avoid interrupting text entry.

In editable contexts:

- `Save changes` and `Close current` still work.
- Other global shortcuts are ignored until focus leaves the input area.

This prevents common conflicts while you type names, SQL, tags, or other form content.

## Customizing shortcuts

You can customize shortcuts from the **Keyboard shortcuts** dialog in the app.

- Changes are saved locally on your device.
- PoliteDB prevents duplicate bindings (one shortcut cannot be assigned to multiple actions).
- Use at least one modifier key (`Cmd/Ctrl`, `Alt`, or `Shift`).
- Press `Esc` to stop recording a new shortcut.
- You can reset one shortcut or reset all shortcuts to defaults.

## Practical tips

- Start by memorizing `Mod+K`, `Mod+T`, `Mod+S`, and `Mod+W`.
- Keep save and close shortcuts easy to reach, since they are used across many workflows.
- If you already use IDE-like bindings, remap PoliteDB to match muscle memory.

Consistent shortcut usage makes PoliteDB feel faster, especially when working across multiple connections and windows.
